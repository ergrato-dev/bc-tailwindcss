¿Qué es el Dark Mode y por qué importa?

El modo oscuro no es solo una tendencia estética — es una característica de accesibilidad que reduce la fatiga ocular en condiciones de baja luminosidad y acomoda a usuarios con fotofobia u otras sensibilidades visuales.

En TailwindCSS, implementar dark mode es sorprendentemente directo gracias a la variante dark: y la estrategia class.

La estrategia class

Con darkMode: 'class', Tailwind aplica las variantes dark: cuando el elemento <html> tiene la clase dark. Esto nos da control programático total sobre el tema.

La estrategia class es superior a media para aplicaciones con toggle porque permite que el usuario elija independientemente de la configuración del sistema operativo.

Implementación del Toggle

El toggle guarda la preferencia en localStorage para persistir entre sesiones. También detecta la preferencia del SO en la primera visita:

function initTheme() {
  const stored = localStorage.getItem('theme')
  const prefersDark = window.matchMedia(
    '(prefers-color-scheme: dark)'
  ).matches

  if (stored === 'dark' || (!stored && prefersDark)) {
    document.documentElement.classList.add('dark')
  }
}

Plugin Typography en Dark Mode

El plugin @tailwindcss/typography incluye la variante especial prose-invert que adapta todos los colores tipográficos para fondos oscuros:

  • Texto principal pasa de gris oscuro a casi blanco
  • Headings adaptan su peso y color
  • Links mantienen su color pero se aclaran
  • Bloques de código cambian fondo y color de sintaxis

Accesibilidad y Contraste

Al implementar dark mode, es crítico verificar que los contrastes siguen cumpliendo WCAG AA (≥4.5:1) en ambos modos. Los colores que pasan en modo claro pueden fallar en modo oscuro y viceversa.

Herramientas recomendadas para verificar:

  1. DevTools de Chrome/Firefox — ícono de contraste en el inspector de color
  2. WebAIM Contrast Checker — verificación manual por par de colores
  3. axe DevTools — auditoría automática de toda la página
