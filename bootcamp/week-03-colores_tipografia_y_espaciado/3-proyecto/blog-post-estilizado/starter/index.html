Cómo el Sistema de Diseño de Tailwind Cambió mi Flujo de Trabajo

Después de años usando CSS arbitrario, descubrí que la consistencia no viene de la disciplina, sino del sistema correcto.

Por Ana García 29 Mar 2026 8 min lectura

El primer día que instalé Tailwind, quise desinstalarlo. Las clases largas en el HTML me parecían un retroceso. Pero le di una semana...

El Problema con el CSS Tradicional

Llevaba 5 años nombrando clases CSS. `.card-header__title--featured`. Cada vez que necesitaba un componente nuevo, pasaba 10 minutos solo decidiendo el nombre. Era "naming fatigue" en estado puro.

"Nombrar cosas es uno de los dos problemas más difíciles en computer science. Tailwind elimina ese problema."

La Escala de Espaciado lo Cambia Todo

Antes usaba valores arbitrarios: `margin: 13px`, `padding: 7px`. Ahora todo es múltiplo de 4. El resultado es coherencia automática.

  • → p-4 = 16px siempre, en cualquier proyecto
  • → Los layouts se "cuadran" solos
  • → Cero debate sobre valores de spacing

⚠ Nota importante

Este patrón solo funciona si usas la escala nativa de Tailwind. Agregar valores custom rompe la coherencia. 


<div style="margin: 13px; padding: 7px;">...</div>

/* Ahora */
<div class="m-3 p-2">...</div>
